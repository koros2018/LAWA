# LAWA 项目日志体系

## 规则

1. **每日进展汇报** — 每天记录开发进展、决策、阻碍
2. **里程碑复盘** — 每个Milestone完成后写复盘文档
3. **对话变更记录** — Ke与达子的所有项目对话、决策、方向变更记录在案
4. **Git版本管理** — 代码稳定后启动Git，方便回滚

## 文件结构

```
Projects/LAWA/memory/
├── README.md          ← 本文件
├── CHANGELOG.md       ← 项目变更日志（决策/方向/规则变更）
├── MILESTONES.md      ← 里程碑状态追踪
├── 2026-05-27.md      ← 每日进展（按日期命名）
├── 2026-05-28.md
└── ...
```

## 日志格式

每日日志需包含：
- **日期 + 当天Sprint**
- **完成的任务**（打勾）
- **关键决策**
- **阻碍/风险**
- **明天计划**
