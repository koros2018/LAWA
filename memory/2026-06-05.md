### 已发现缺陷

| 严重度 | 问题 | 位置 |
|--------|------|------|
| 🔴 高 | 注册缺少密码强度校验（`123`可通过） | auth.py |
| 🔴 高 | 注册缺少 `learn_lang` 字段 | auth.py |
| 🟡 中 | `/community/guilds` 返回404 | community.py缺失guild路由 |
| 🟡 中 | 任务创建需要 `publisher_id`（前端未传） | task.py |
| 🟡 中 | 公会创建需要 `user_id`（前端未传） | rpg.py |
| 🟡 中 | 评估answer/report需要额外字段 | assessment.py |
| 🟢 低 | 金币balance需要user_id路径参数（应从token获取） | coin.py |

### 总结
核心模块（认证/评估/RPG/LLM/金币/任务）全部可用，主要缺陷集中在**输入校验**和**字段缺失**。前端对接时需要确保传递 `publisher_id`、`user_id`、`learn_lang` 等必填字段。
